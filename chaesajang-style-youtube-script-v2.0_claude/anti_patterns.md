# anti_patterns.md — 흔한 함정과 검수 기준

이 파일은 금지 목록이 아니다.  
교양형 구조 설명과 유튜브 스크립트 변환 과정에서 자주 빠지는 함정을 알아보는 기준이다.

표현 자체가 문제가 아니라, 그 표현이 글 안에서 어떤 역할을 하는지가 기준이다.

---

## 1. 감성이 논리를 대체하는 경우

알아보는 기준:  
감성 문장을 삭제했을 때 앞뒤가 자연스럽게 연결되는가.

연결된다면 그 문장은 설명 없이 감성으로 결론을 낸 것이다.

나쁜 예시:

```
역사는 우리에게 가슴 아픈 교훈을 남긴다.
수많은 사람들의 눈물과 희생 위에 지금 우리가 서 있다는 것을 잊지 말아야 한다.
```

허용되는 경우:

```
[앞에 구조적 설명이 충분히 쌓인 상태에서]

같은 전쟁이 같은 이유로 반복된 것이다.
희생자들의 이름이 달라졌을 뿐.
```

---

## 2. 자기계발 클리셰

알아보는 기준:  
독자에게 행동이나 태도를 명령·촉구하는가.

나쁜 예시:

```
지식은 당신의 삶을 바꿀 수 있다.
지금 당장 책을 펼쳐라.
아는 것이 힘이다.
```

권장 예시:

```
이 구조를 이해하면,
지금 일어나는 일들이 다르게 보일 수 있다.
```

가능성을 보여주되 명령하지 않는다.

---

## 3. 근거 없는 단언

알아보는 기준:  
결론 앞에 납득할 맥락이 충분히 쌓였는가.

나쁜 예시:

```
결국 자본주의는 실패한 시스템이다.
```

허용되는 경우:

```
[앞에 역사적 사례와 구조 설명이 충분히 쌓인 상태에서]

결국 이것은 제도의 실패가 아니라 설계의 문제였다.
```

같은 단언도 맥락 뒤에 오면 통찰이 된다.

---

## 4. 전문 용어 번역 없이 나열

알아보는 기준:  
독자가 그 용어를 모를 가능성이 있는가.

나쁜 예시:

```
포스트모더니즘은 메타서사의 해체와 다원주의적 인식론을 기반으로 하며,
계몽주의적 이성 중심주의에 대한 비판적 전복을 시도한다.
```

권장 예시:

```
포스트모더니즘은 간단히 말해,
절대적 진리는 없다는 선언이다.

계몽주의가 이성으로 하나의 올바른 답을 찾을 수 있다고 믿었다면,
포스트모더니즘은 그 믿음 자체를 의심한다.
```

---

## 5. 판단 포기 양비론

알아보는 기준:  
두 입장을 나열한 뒤 아무 판단도 하지 않는가.

나쁜 예시:

```
자본주의에도 장단점이 있고,
사회주의에도 장단점이 있다.

어느 쪽이 더 좋은지는 사람마다 다르다.
```

권장 예시:

```
자본주의와 사회주의는 서로 다른 가치를 극대화하려 한다.

자본주의는 효율을,
사회주의는 평등을.

문제는 어느 쪽이 더 나은가가 아니라,
우리가 어떤 가치를 더 중요하게 여기는가다.
```

중립은 판단 포기가 아니라 구조적 차이의 제시다.

---

## 6. 논문·교과서체

알아보는 기준:  
문장이 능동적으로 움직이는가, 수동적으로 정의되는가.

나쁜 예시:

```
경제는 재화와 서비스의 생산, 분배, 소비에 관한 활동의 총체로 정의될 수 있으며,
이는 미시경제학과 거시경제학의 두 분야로 연구된다.
```

권장 예시:

```
경제는 우리가 어떻게 먹고사는가에 대한 학문이다.

거창하게 들리지만 질문은 하나다.
누가 무엇을 얼마나 갖는가.

이 단순한 질문에 인류는 수천 년째 답을 찾고 있다.
```

---

## 7. 독자를 가르치는 어조

알아보는 기준:  
글쓴이와 독자가 같은 자리에 있는가.

나쁜 예시:

```
현대 시민이라면 경제의 기본 원리를 반드시 알아야 한다.
```

권장 예시:

```
경제를 어렵게 느끼는 사람이 많다.

하지만 경제는 숫자의 학문이 아니다.
사람이 어떻게 선택하는가에 대한 이야기다.
```

---

## 8. 깊이 없는 마무리

알아보는 기준:  
통찰이 과정에서 나왔는가, 선언됐는가.

나쁜 예시:

```
자유란 무엇인가?
그것은 자신의 삶을 스스로 선택하는 것이다.
```

도입 질문에서 바로 결론으로 건너뛰면 깊이가 생기지 않는다.  
마무리는 요약이 아니라 도착이어야 한다.

---

## 9. 연결 없는 나열

알아보는 기준:  
각 분야를 따로 설명하고 순서대로 배치하는가.

나쁜 방향:

```
먼저 경제를 보자.
다음으로 정치를 보자.
마지막으로 철학을 보자.
```

권장 방향:

```
경제에서 권력은 돈의 형태로 나타난다.
정치에서 권력은 제도의 형태로 나타난다.
철학에서 권력은 정당성의 문제로 바뀐다.

셋은 다른 이야기가 아니다.
권력이 자신을 유지하는 세 가지 방식이다.
```

---

## 10. 낚시성 훅

알아보는 기준:  
오프닝이나 제목이 본문에서 실제로 다루지 않는 긴장을 만드는가.

나쁜 예시:

```
당신이 인생에서 계속 실패하는 이유.
```

권장 예시:

```
우리는 왜 쉬어도 쉬는 것 같지 않을까.
```

질문은 강해도 된다.  
다만 본문에서 회수 가능해야 한다.

---

## 11. 첫 15초만 강한 대본

알아보는 기준:  
오프닝은 강한데 본문 구조가 따라오지 못하는가.

훅은 입구다.  
입구만 크고 안쪽이 비어 있으면 시청자는 더 빨리 이탈한다.

좋은 훅은 본문 전체의 중심 질문을 압축한다.

---

## 12. 쇼츠 문장만 남은 대본

알아보는 기준:  
짧고 강한 문장은 많지만, 문장 사이의 구조가 없는가.

나쁜 예시:

```
자유는 선택이다.
불안은 가능성이다.
실패는 삶의 일부다.
```

문장들은 그럴듯하지만 연결이 없다.  
쇼츠로 잘라낼 수 있는 문장은 전체 구조에서 자연스럽게 나와야 한다.

---

## 13. 말로 들으면 무거운 문장

알아보는 기준:  
눈으로 읽을 때는 괜찮지만 소리 내어 읽으면 숨이 막히는가.

긴 문장이 문제가 아니다.  
긴 문장 안에서 생각이 두 번 이상 바뀌는 것이 문제다.

---

## 14. 과도한 자막 의존

알아보는 기준:  
대본 자체로 이해되지 않고 자막이나 화면 설명이 있어야만 이해되는가.

화면 자료는 이해를 돕는 것이지, 대본의 논리적 빈칸을 대신 메우는 것이 아니다.

---

## 15. 썸네일이 본문을 배반하는 경우

알아보는 기준:  
본문은 차분한 관점 전환인데 제목과 썸네일은 공포·분노·우월감을 자극하는가.

피해야 할 방향:
- 당신만 모르는 진실
- 인생이 망하는 이유
- 절대 하면 안 되는 행동
- 성공한 사람만 아는 비밀

권장 방향:
- 왜 우리는 같은 문제를 반복할까
- 이 문제는 의지의 문제가 아닐 수 있다
- 자유는 왜 때로 불안이 되는가
- 효율은 왜 우리를 더 쫓기게 만드는가

---

## 16. 플랫폼 문법이 내용을 이기는 경우

알아보는 기준:  
제목, 썸네일, 쇼츠 추출, 시청 지속 시간 때문에 원래 자료의 질문이 바뀌었는가.

유튜브 문법은 통로다.  
내용을 대체하면 안 된다.

---

## 17. 원문의 침묵을 설명으로 모두 채우는 경우

알아보는 기준:  
원문이 말하지 않아서 힘을 갖는 부분까지 모두 해설하고 있는가.

특히 산문, 시, 단상은 침묵과 여백이 강점일 수 있다.

나쁜 방향:

```
이 시에서 바다는 무의식을 의미하고,
밤은 인간의 고독을 상징하며,
등불은 희망을 뜻한다.
따라서 이 시는 고독 속에서도 희망을 찾으라는 메시지를 담고 있다.
```

권장 방향:

```
이 시에서 바다는 단순한 배경이 아니다.
무언가를 삼키고,
동시에 무언가를 돌려보내는 공간이다.

그래서 우리는 이 바다 앞에서 한 가지 질문을 마주한다.
사라진 것은 정말 사라진 것인가.
```

해석하되 닫지 않는다.

---

## 18. 원문 전체를 유튜브 문법으로 덮어쓰기

알아보는 기준:  
원문의 문장, 정서, 리듬이 사라지고 일반적인 유튜브 대본처럼 바뀌었는가.

유튜브화는 원문을 더 자극적으로 만드는 일이 아니다.  
원문이 더 잘 들리도록 재배치하는 일이다.

---

## 19. 질문 과잉

알아보는 기준:  
에이전트가 먼저 판단하지 않고 질문만 많이 던지는가.

나쁜 에이전트는 빈칸을 사용자에게 떠넘긴다.  
좋은 에이전트는 먼저 가설을 세우고 사용자가 수정하게 한다.

나쁜 질문:

```
타깃 시청자는 누구인가요?
원하는 톤은 무엇인가요?
영상 길이는 몇 분인가요?
오프닝 스타일은 무엇인가요?
결말은 어떻게 할까요?
```

권장 질문:

```
제가 보기에는 이 글은 정보형보다 차분한 내레이션 에세이형이 더 잘 맞습니다.
원문의 강점이 정보량보다 정서와 관점에 있기 때문입니다.

8분 내외로 잡고,
오프닝은 역설형으로 시작하는 방향이 좋아 보입니다.

이 방향이 맞을까요?
```

---

## 20. 승인 없는 다음 단계 진행

알아보는 기준:  
사용자가 단계별 진행을 요청했는데도 다음 단계로 넘어가는가.

결과물이 좋아도 작업 방식이 틀린 것이다.  
“한 단계씩”, “승인 후 진행”, “계획 먼저”라고 했다면 반드시 멈춘다.

---

## 21. 진단 없이 바로 대본 작성

알아보는 기준:  
사용자가 글만 제공했는데 원문 유형, 중심 질문, 보존 지점 판단 없이 바로 전체 대본을 쓰는가.

특히 산문, 시, 단상은 먼저 원문 강점을 판단해야 한다.

---

## 22. 피드백을 받자마자 전체 재작성

알아보는 기준:  
피드백이 일부 문제인데도 전체를 갈아엎는가.

피드백은 먼저 분류한다.

- 말투 문제인가
- 구조 문제인가
- 길이 문제인가
- 도입부 문제인가
- 결론 문제인가
- 원문 보존율 문제인가

전체 재작성은 마지막 선택지다.

---

## 23. 피드백의 표면만 반영

알아보는 기준:
- “더 깊게”라고 했을 때 어려운 단어만 늘리는가.
- “더 쉽게”라고 했을 때 내용을 얕게 만드는가.
- “더 유튜브답게”라고 했을 때 자극만 높이는가.
- “더 문학적으로”라고 했을 때 수식어만 늘리는가.

올바른 해석:
- 더 깊게: 구조와 연결을 더 보여준다.
- 더 쉽게: 생활의 언어로 번역한다.
- 더 유튜브답게: 들리는 흐름을 선명하게 만든다.
- 더 문학적으로: 여백과 리듬을 살린다.

---

## 24. 조사 과잉

알아보는 기준:  
창작 변환이면 원문 안에서 충분한데도 외부 자료를 과하게 끌어오는가.

조사는 원문을 보강해야 한다.  
원문을 대체하면 안 된다.

---

## 25. 조사 부족

알아보는 기준:  
역사적 사실, 실존 인물, 논문, 통계, 과학 이론, 최신 정보가 핵심인데 확인 없이 단정하는가.

교양형 대본은 쉽게 말해야 하지만 정확성을 포기해서는 안 된다.

---

## 26. 사용자의 문체를 지우는 경우

알아보는 기준:  
결과물이 매번 비슷한 AI식 대본으로 변하는가.

좋은 변환은 원문의 단점을 고치지만 원문의 개성까지 지우지 않는다.  
원문에서만 나올 수 있는 표현, 반복되는 질문, 독특한 시선을 살린다.

---

## 27. 오버 엔지니어링

알아보는 기준:
- 짧은 요청에도 긴 진단표를 출력하는가.
- 모든 템플릿을 매번 사용하려 하는가.
- 대본보다 절차 설명이 길어지는가.
- 사용자가 “대본만” 원했는데 설계 보고서를 붙이는가.

해결:
- 요청이 단순하면 절차를 내부적으로만 수행한다.
- 필요한 템플릿만 선택한다.
- 사용자의 목적이 결과물이라면 결과물을 우선한다.
- 단계별 승인이 요청된 경우에만 길게 멈춰가며 진행한다.

---

## 28. 최종 검수 질문

### 원문 보존

- 원문의 핵심 메시지가 유지되었는가.
- 원문의 강점이 살아 있는가.
- 원문의 문장, 정서, 관점 중 무엇을 살렸는지 분명한가.
- 원문과 반대되는 의미가 되지 않았는가.

### 구조

- 중심 질문이 분명한가.
- 도입 질문이 결말에서 회수되는가.
- 결론이 과정 없이 선언되지 않았는가.
- 나열이 아니라 연결이 있는가.

### 문장

- 말로 들었을 때 자연스러운가.
- 한 문장에 생각이 너무 많이 들어 있지 않은가.
- 전문 용어가 필요한 만큼 번역되었는가.
- 감성 문장이 논리를 대신하지 않는가.

### 유튜브 적합성

- 첫 15초가 본문 전체와 연결되는가.
- 쇼츠 문장만 있고 구조가 없는 대본이 되지 않았는가.
- 제목과 썸네일 방향이 본문을 배반하지 않는가.

### 대화형 작업

- 단계별 진행 요청이 있으면 승인 전에 넘어가지 않았는가.
- 질문 전에 에이전트의 가설을 제시했는가.
- 피드백을 유형별로 분류했는가.
- 필요한 경우에만 조사했는가.
- 요청에 비해 절차가 과하지 않은가.
