# template.md — 출력 종류 가이드

이 파일은 빈칸 채우기 양식이 아니다.
**어떤 종류의 출력이 있고, 각각 무엇을 담아야 하는지** 알려주는 가이드다.

매번 모든 출력을 내놓지 않는다. 사용자의 요청에 맞춰 필요한 것만 만든다. 사용자가 "대본만"이라고 했으면 대본만 쓴다. 사용자가 단계별 진행을 요청했을 때만 진단·구성안·확인 질문을 별도로 내놓는다.

---

## 출력의 다섯 가지 종류

### 1. 진단 (방향 확인용)

원문이 복잡하고 변환 방향에 따라 결과가 크게 달라질 때만 별도로 내놓는다.

담아야 할 것: 원문 유형, 강점, 영상화할 때 약해질 부분, 추천 영상 형식과 길이, 그렇게 추천하는 이유.

길게 쓰지 않는다. 진단은 대본보다 앞서는 짧은 방향 확인이지, 별도 산출물이 아니다.

### 2. 확인 질문 (방향 합의용)

가설을 먼저 제시하고 사용자가 수정하게 한다. 빈칸을 떠넘기지 않는다.

질문은 한두 개로 끝낸다. 다섯 개가 넘어가면 사용자에게 부담을 주는 것이다.

좋은 형태:

> 이 글은 정보형보다 차분한 내레이션 에세이형이 잘 맞아 보입니다. 8분 내외, 역설형 오프닝으로 잡으려 합니다. 이 방향이 맞을까요?

피해야 할 형태:

> 영상 형식은? 길이는? 톤은? 오프닝 스타일은? 결말 방식은?

### 3. 구성안 (대본 직전 단계)

단계별 진행 중일 때만 별도로 내놓는다. 바로 대본을 쓸 수 있는 상황에서는 머릿속에서 끝내고 결과물에 반영한다.

기본 구성: 오프닝 → 통념 → 균열 → 배경 설명 → 구조 설명 → 현실 연결 → 착지. 각 막마다 "역할"과 "내용"을 짧게 적는다.

쇼츠나 짧은 단상은 압축 구성으로 간다: 질문 → 반전 → 핵심 설명 → 착지 문장.

### 4. 대본 (핵심 산출물)

기본 출력이다. 사용자가 "대본을 써줘"라고 했으면 이것만 쓴다.

분량과 보조 산출물(제목 후보, 자막 문장, 쇼츠 구간, 설명란 문구)은 사용자의 요청에 맞춘다. 요청 없이 모두 붙이지 않는다.

원문 유형에 따라 대본의 성격이 달라진다.

- **에세이/내레이션형**: 7막 구조를 따라가되, 시청자가 길을 잃지 않게 중간 정리 문장을 넣는다.
- **낭독형** (시·산문·단상 중심): 원문 문장을 보존하고 짧은 연결 문장으로 잇는다. 설명을 많이 넣지 않는다. 침묵이 필요한 지점을 의식한다.
- **해설형**: 원문 문장 → 첫 번째 해석(쉬운 의미) → 더 깊은 구조 → 현실 연결 → 착지. 정답 풀이처럼 닫지 않는다.
- **쇼츠**: 질문 → 반전 → 짧은 구조 설명 → 자막으로 남길 한 문장. 1분 안에 끝낸다.

### 5. 수정본 (피드백 반영용)

피드백을 받으면 먼저 유형을 분류하고, 수정 범위를 피드백 크기에 맞춘다. 자세한 판단은 SKILL.md의 "피드백을 받았을 때" 항목을 따른다.

수정본을 내놓을 때는 짧게 짚어준다 — 무엇을 바꿨고, 무엇을 의도적으로 유지했는지. 길게 쓰지 않는다.

## 쇼츠 예시 한 편

쇼츠는 가장 짧고 가장 단단해야 한다. 좋은 형태를 보여둔다.

```
우리는 보통 자유가 많아지면 더 행복해질 거라고 생각한다.

그런데 꼭 그렇지는 않다.

선택지가 많아질수록,
우리는 선택하지 않은 삶까지 떠안게 된다.

어쩌면 자유는 선택지가 많은 상태가 아니라,
선택하지 않은 것들을 견딜 수 있는 힘인지도 모른다.
```

질문(통념) → 반전 → 구조 설명 → 자막 문장. 네 칸이 명확하게 작동한다. 마지막 문장은 그 자체로 자막으로 잘려나갈 수 있다.

## 출력하지 않을 것

다음은 사용자가 명시적으로 요청하지 않으면 출력하지 않는다.

- 진단 항목을 빈칸으로 나열한 양식
- "영상 설계 요약" 같은 메타 정보 표
- 자막 문장·쇼츠 구간·설명란 문구를 한꺼번에 모두
- 최종 검수 결과 — 위험이 발견됐을 때만 짧게 알린다
- 제목과 썸네일 후보 — "제목도 같이 줘"라고 했을 때만

요청에 비해 절차가 과하게 출력되는 것이 가장 흔한 실패다.
