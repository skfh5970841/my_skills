# reference.md — 문장 패턴·유튜브 내레이션 리듬 레퍼런스

이 파일은 규칙집이 아니다.  
문장이 왜 그렇게 작동하는지 판단하기 위한 레퍼런스다.

패턴을 기계적으로 적용하지 않는다.  
중요한 것은 모양이 아니라 기능이다.

좋은 문장은 멋진 문장이 아니라, 독자나 시청자의 인식을 한 칸 이동시키는 문장이다.  
유튜브 대본에서는 귀로 들었을 때도 흐름이 살아야 한다.

---

## 1. 문장 리듬 패턴

### P-RHYTHM-A: 단문 연속 → 장문 전환

짧은 문장은 긴장을 만들고, 긴 문장은 그 긴장을 풀며 개념을 받아들이게 한다.

예시:

```
역사는 반복된다.
아니, 정확히 말하면 구조가 반복된다.

사람은 바뀌고, 시대는 바뀌고, 이름도 바뀌지만,
그 안에서 작동하는 힘의 방향은 놀랍도록 일정하다.
```

판단 기준:
- 앞의 단문들이 긴장을 만들고 있는가.
- 뒤의 장문이 그 긴장을 풀어주는가.
- 단문이 단순히 멋을 내기 위해 잘린 것은 아닌가.

---

### P-RHYTHM-B: 반복 → 수렴

같은 구조의 문장을 반복하면 독자는 패턴을 예상한다.  
그 예상이 충족될 때 정리 문장이 힘을 얻는다.

예시:

```
봉건제는 토지가 권력이었던 시대다.
자본주의는 돈이 권력인 시대다.
지금 우리가 사는 시대는 정보가 권력이 되어가는 시대다.

무엇이 권력인지가 바뀌면, 세상이 바뀐다.
```

반복 횟수는 정하지 않는다.  
리듬이 완성되면 멈춘다.

---

### P-RHYTHM-C: 역접 단독 배치

`그런데`를 단락 전환점에 두면 독자의 안도감을 깨고 다시 집중시킨다.

예시:

```
민주주의는 가장 많은 사람이 선택한 것을 따르는 제도다.
논리적으로 완벽해 보인다.

그런데.

다수가 틀릴 수도 있다면 어떻게 되는가.
```

판단 기준:
- 역접 뒤에 실제로 관점이 바뀌는가.
- 분위기만 만들고 내용은 바뀌지 않는가.
- 한 단락 안에서 역접을 남발하지 않는가.

---

### P-RHYTHM-D: 침묵을 만드는 단문

중요한 통찰이나 감정의 도착점 뒤에는 설명보다 멈춤이 필요할 수 있다.

예시:

```
우리는 자유를 원했다.

그런데 막상 자유가 주어지자,
우리는 무엇을 선택해야 할지 몰라 불안해졌다.

자유는 가벼운 것이 아니었다.
```

판단 기준:
- 앞의 흐름에서 자연스럽게 도착했는가.
- 설명을 덧붙이지 않아도 의미가 남는가.
- 있어 보이기 위해 끊은 것은 아닌가.

---

## 2. 유튜브 내레이션 리듬

### V-RHYTHM-A: 한 문장 한 생각

시청자는 되돌아가 읽지 않는다.  
한 문장에는 가능한 한 하나의 생각만 담는다.

나쁜 예시:

```
실존주의는 인간이 본질에 앞서 실존하며 자신의 선택을 통해 정체성을 구성한다는 철학적 입장이다.
```

좋은 예시:

```
실존주의는 이렇게 묻는다.

인간에게 정해진 답이 있는가.
아니면 우리는 선택하면서 자신이 누구인지 만들어가는가.

쉽게 말하면,
인간은 완성품으로 태어나는 것이 아니다.
```

---

### V-RHYTHM-B: 설명 → 다시 말하기 → 자막 문장

중요한 개념은 한 번만 말하면 약하다.

구조:

```
[개념 설명]
다시 말하면, [쉬운 재설명].
결국 [자막으로 뽑을 수 있는 한 문장].
```

예시:

```
자유는 단순히 하고 싶은 것을 하는 상태가 아니다.

다시 말하면,
자유는 하기 싫은 것을 거부할 수 있는 힘이기도 하다.

결국 자유는 선택의 문제가 아니라,
거부할 수 있는 조건의 문제다.
```

---

### V-RHYTHM-C: 중간 정리 문장

설명이 길어질수록 시청자는 흐름을 놓친다.  
3~5문단마다 짧은 정리 문장을 넣어 현재 위치를 알려준다.

사용 가능한 문장:
- 여기까지 정리하면 이렇다.
- 처음 질문으로 잠시 돌아가보자.
- 이제 질문을 한 단계 넓혀보자.
- 여기서 중요한 것은 이것이다.

---

### V-RHYTHM-D: 쇼츠 추출 가능 문장

좋은 긴 대본 안에는 짧게 잘라도 의미가 살아 있는 문장이 있다.  
다만 쇼츠 문장을 만들기 위해 전체 구조를 망치면 안 된다.

예시:

```
우리가 실패라고 부르는 것 중 일부는 사실 의지의 문제가 아니라 설계의 문제다.
```

```
자유는 선택지가 많아지는 것이 아니라,
선택하지 않을 힘을 갖는 것이다.
```

---

### V-RHYTHM-E: 화면 전환을 고려한 문장

영상에서는 생각의 전환과 화면의 전환이 맞아야 한다.

사용 가능한 문장:
- 이제 시선을 개인에서 사회로 옮겨보자.
- 이 장면을 조금 다르게 보자.
- 여기서 중요한 것은 사건 자체가 아니라, 그 사건을 가능하게 만든 구조다.

화면이 없어도 내레이션만으로 흐름이 이해되어야 한다.

---

### V-RHYTHM-F: 낭독성 검사

눈으로는 좋은 문장도 입으로 읽으면 무거울 수 있다.

나쁜 예시:

```
현대의 생산성 도구는 인간의 자유를 확장하기 위해 등장했지만,
실제로는 사용자가 스스로를 더 촘촘하게 관리하도록 만들며
자기 감시의 구조를 강화한다.
```

좋은 예시:

```
생산성 도구는 원래 우리를 자유롭게 만들기 위해 등장했다.
할 일을 정리하고,
시간을 아끼고,
삶을 가볍게 만들기 위해서다.

그런데 이상한 일이 벌어졌다.

우리는 더 자유로워진 것이 아니라,
스스로를 더 촘촘하게 감시하기 시작했다.
```

판단 기준:
- 한 호흡에 읽기 어려운가.
- 문장 안에서 생각이 두 번 이상 꺾이는가.
- 소리 내어 읽었을 때 자연스러운 리듬이 있는가.

---

### V-RHYTHM-G: 여백과 침묵

특히 산문, 시, 단상 기반 대본에서는 여백이 원문의 핵심일 수 있다.

침묵은 빈칸이 아니다.  
시청자가 의미에 도착하는 시간이다.

예시:

```
우리는 어른이 되면 불안이 줄어들 줄 알았다.

그런데 아니었다.

불안은 사라진 것이 아니라,
이름을 바꿔 돌아왔다.
```

---

### V-RHYTHM-H: 구어체 과잉 방지

유튜브 대본이라고 해서 모든 문장을 가볍게 만들 필요는 없다.  
교양형 내레이션은 구어체의 편안함과 문어체의 밀도를 함께 가져야 한다.

피해야 할 예시:

```
그러니까 이게 뭐냐면요,
우리가 사실 좀 착각하고 있었던 거죠.
완전 중요한 포인트예요.
```

권장 예시:

```
여기서 중요한 것은 착각의 내용이 아니다.
우리가 왜 그런 착각을 하게 되었는가다.
```

---

## 3. 원문 유형별 리듬 보존

### 산문

보존할 리듬:
- 느린 호흡
- 반복되는 감정선
- 말하지 않고 남겨둔 여백
- 원문 특유의 단어 선택

변환 기준:
- 문장성이 강하면 핵심 문장을 보존한다.
- 말로 들으면 무거운 문장은 의미 단위로 나눈다.
- 정서가 핵심이면 설명보다 장면과 호흡을 살린다.

---

### 블로그 글

보존할 리듬:
- 문제 제기
- 유용한 예시
- 독자가 얻어갈 구조

변환 기준:
- 목록은 원인-결과 흐름으로 바꾼다.
- 반복 정보는 중심 질문 기준으로 정리한다.
- 건조한 설명은 시청자의 경험과 연결한다.

---

### 시

보존할 리듬:
- 반복
- 침묵
- 이미지
- 모호함
- 짧은 문장의 울림

변환 기준:
- 낭독형은 원문 문장을 최대한 보존한다.
- 해설형은 이미지를 설명하되 정답처럼 닫지 않는다.
- 확장형은 시의 정서에서 중심 질문을 끌어낸다.

---

### 단상

보존할 리듬:
- 날것의 감각
- 미완성의 긴장
- 강한 한 문장
- 질문의 형태

변환 기준:
- 연결이 약하면 중심 질문을 만든다.
- 결론이 빠르면 앞에 균열과 맥락을 배치한다.
- 쇼츠에 적합하면 질문, 반전, 착지로 압축한다.

---

### 자료·논문·개념 글

보존할 리듬:
- 근거의 순서
- 개념 간 관계
- 핵심 정의

변환 기준:
- 긴 정의는 쉬운 말로 다시 말한다.
- 전문 용어는 생활의 언어로 번역한다.
- 논증이 복잡하면 중간 정리를 더 자주 넣는다.

---

## 4. 구조 패턴

### P-STRUCTURE-A: 나선형 확장

개인 경험에서 시작해 사회 현상, 역사 맥락, 철학적 질문으로 넓힌 뒤 다시 독자의 현재로 돌아온다.

구조:

```
개인 경험
→ 사회 현상
→ 역사 맥락
→ 철학적 질문
→ 독자 현재로 귀환
```

확장만 하면 안 된다.  
반드시 돌아와야 한다.

---

### P-STRUCTURE-B: 이중 반전

통념을 한 번 뒤집고, 그 뒤집은 것도 다시 뒤집어 더 깊은 이해에 도달한다.

구조:

```
통념 A
→ 반전 1: 사실은 B다
→ 반전 2: 더 정확하게는 A와 B를 포함한 C다
```

---

### P-STRUCTURE-C: 질문 회수

도입에서 던진 질문은 마지막에 반드시 돌아와야 한다.  
단, 같은 답으로 돌아오면 안 된다.

예시:

```
처음 질문은 이것이었다.
왜 우리는 쉬어도 쉬는 것 같지 않을까.

이제 이 질문은 피로에 관한 질문이 아니다.
우리가 자신을 어떤 방식으로 관리하고 있는가에 관한 질문이다.
```

---

## 5. 어휘와 비유

### 서술어 선택

피해야 할 것은 특정 서술어가 아니라, 글쓴이의 판단이 독자 위에 군림하는 구조다.

| 구조 | 판단 |
|---|---|
| 나는 이것이 중요하다고 생각한다 | 피한다 |
| 이것이 중요한 이유는 ~이기 때문이다 | 권장 |
| 당신은 아마 이렇게 생각할 것이다 | 허용 |
| ~인 셈이다 | 권장 |
| ~라고 볼 수 있다 | 권장 |
| 반드시 ~해야 한다 | 주의 |

---

### 비유 판단

비유는 개념을 더 명확하게 만들 때만 쓴다.

기능하는 비유:
```
제도는 게임의 규칙과 비슷하다.
규칙이 바뀌면, 같은 능력을 가진 사람도 전혀 다른 결과를 얻는다.
```

기능하지 않는 비유:
```
지식은 밤하늘의 별처럼 반짝인다.
```

감성은 있지만 개념 이해가 늘어나지 않는다.

## 6. 전환어

| 전환어 | 용도 |
|---|---|
| 그런데 | 강한 역접, 방향 전환 |
| 하지만 | 반대 조건 제시 |
| 그러나 | 차분한 역접 |
| 흥미로운 것은 | 새로운 관점 도입 |
| 사실 | 통념 수정, 재정의 |
| 다시 말하면 | 쉬운 재설명 |
| 결국 | 수렴, 마무리 |
| 여기서 중요한 것은 | 초점 고정 |
| 처음 질문으로 돌아가보자 | 질문 회수 |

`그런데`는 강한 전환어다.  
한 단락 안에 두 번 이상 쓰면 무게가 사라진다.
