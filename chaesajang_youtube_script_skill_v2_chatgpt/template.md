# template.md — 출력 템플릿

이 파일은 공식이 아니다.  
필요한 상황에서 선택해 쓰는 출력 형식 모음이다.

모든 템플릿을 매번 출력하지 않는다.  
사용자의 요청이 단순하면 결과물만 제공한다.  
사용자가 단계별 진행을 요청하거나 방향 확인이 필요할 때만 진단·질문·구성안 템플릿을 사용한다.

---

## 1. 원문 진단 템플릿

```
# 원문 진단

- 원문 유형:
- 핵심 주제:
- 숨어 있는 중심 질문:
- 반드시 살릴 문장/정서:
- 영상에서 약해질 수 있는 부분:
- 추천 영상 형식:
- 추천 영상 길이:
- 추천 오프닝 방식:

## 판단

이 원문은 [유형]에 가깝습니다.
강점은 [문장/정서/구조/관점/정보]입니다.

따라서 대본은 [추천 형식]으로 가는 편이 좋습니다.
이유는 [원문 강점]을 살리면서도 [위험 요소]를 보완할 수 있기 때문입니다.
```

짧게 쓴다.  
진단은 대본보다 앞서는 방향 확인용 산출물이다.

---

## 2. 가설형 질문 템플릿

```
# 확인 질문

제가 보기에는 이 글은 [추천 방향]이 가장 잘 맞습니다.
이유는 [원문 근거] 때문입니다.

다만 확정 전에 아래를 확인하고 싶습니다.

1. 영상 형식은 [추천 형식]으로 잡아도 될까요?
   - 이유:

2. 예상 길이는 [추천 길이]가 적합해 보입니다. 괜찮을까요?
   - 이유:

3. 원문의 문장은 [보존율] 정도로 살리는 편이 좋아 보입니다. 이 방향이 맞을까요?
   - 이유:

4. 오프닝은 [질문형/역설형/장면형]이 좋아 보입니다. 괜찮을까요?
   - 예시:

5. 결말은 [여운형/정리형/현실 연결형] 중 [추천]이 적합해 보입니다. 이 방향으로 갈까요?
```

질문은 최대 5개.  
이미 답한 내용은 다시 묻지 않는다.

---

## 3. 영상 방향 확정 템플릿

```
# 영상 방향 확정

- 영상 형식:
- 예상 길이:
- 중심 질문:
- 핵심 메시지:
- 원문 보존율:
- 말투:
- 오프닝 방식:
- 결말 방식:
- 시청자 가정:
- 주의할 점:

## 확정된 방향

이 영상은 [중심 질문]을 중심으로 전개합니다.
원문의 [강점]은 살리고, 영상에서 약해질 수 있는 [위험 요소]는 [보완 방식]으로 처리합니다.
```

단계별 진행 중이면 여기서 멈추고 승인받는다.

## 4. 오프닝 후보 템플릿

```
# 오프닝 후보

## A. 질문형

[오프닝 문장]

적합한 이유:
주의할 점:

---

## B. 역설형

[오프닝 문장]

적합한 이유:
주의할 점:

---

## C. 장면형

[오프닝 문장]

적합한 이유:
주의할 점:

---

## 추천

이 자료에는 [유형]이 가장 적합합니다.
이유는 [자료의 핵심]이 [해당 오프닝의 작동 방식]과 맞기 때문입니다.
```

## 5. 구성안 템플릿

```
# 영상 구성안

## 1막. 오프닝
역할: 시청자를 질문 앞에 세운다.
내용:

## 2막. 통념
역할: 사람들이 보통 어떻게 생각하는지 보여준다.
내용:

## 3막. 균열
역할: 통념으로 설명되지 않는 지점을 만든다.
내용:

## 4막. 배경 설명
역할: 필요한 개념, 역사, 심리, 사회 구조를 놓는다.
내용:

## 5막. 구조적 연결
역할: 흩어진 요소를 하나의 구조로 묶는다.
내용:

## 6막. 현실 연결
역할: 시청자의 삶이나 판단으로 돌아온다.
내용:

## 7막. 착지
역할: 처음 질문에 더 깊어진 답으로 돌아온다.
내용:
```

### 압축 구성안

쇼츠나 짧은 단상에는 아래를 쓴다.

```
# 압축 구성안

1. 질문:
2. 반전:
3. 핵심 설명:
4. 착지 문장:
```

## 7. 대본만 출력 템플릿

```
# 전체 대본

## 오프닝

[내레이션]

## 본론

[내레이션]

## 마무리

[내레이션]
```

불필요한 분석, 제목 후보, 검수표를 붙이지 않는다.

---

## 8. 쇼츠 대본 템플릿

```
# 쇼츠 대본

## 질문

[시청자가 자기 문제로 느낄 질문]

## 반전

[통념을 뒤집는 문장]

## 설명

[짧은 구조 설명]

## 착지

[자막으로 남길 수 있는 한 문장]
```

예시:

```
우리는 보통 자유가 많아지면 더 행복해질 거라고 생각한다.

그런데 꼭 그렇지는 않다.

선택지가 많아질수록,
우리는 선택하지 않은 삶까지 떠안게 된다.

어쩌면 자유는 선택지가 많은 상태가 아니라,
선택하지 않은 것들을 견딜 수 있는 힘인지도 모른다.
```

## 9. 낭독형 대본 템플릿

시, 산문, 단상처럼 원문 문장성이 중요한 경우 사용한다.

```
# 낭독형 대본

## 영상 설계

- 원문 보존율:
- 낭독 톤:
- 화면 분위기:
- 여백이 필요한 지점:

---

## 대본

[원문 보존 문장]

[짧은 연결 문장]

[원문 확장 문장]

[침묵 또는 화면 전환이 필요한 지점]

[착지 문장]
```

주의:
낭독형은 설명을 많이 넣지 않는다.

---

## 10. 해설형 대본 템플릿

```
# 해설형 대본

## 1. 원문 또는 핵심 문장 제시

[원문 문장]

## 2. 첫 번째 해석

[가장 쉬운 의미 설명]

## 3. 더 깊은 구조

[철학, 심리, 사회 구조와 연결]

## 4. 현실 연결

[시청자의 삶과 연결]

## 5. 착지

[처음 문장을 다시 다르게 보이게 만드는 결론]
```

주의:
해설형은 원문을 정답 풀이처럼 닫지 않는다.

---

## 11. 피드백 반영 템플릿

```
# 피드백 반영 방식

받은 피드백:
-

피드백 유형:
- 말투 수정 / 구조 수정 / 길이 조정 / 도입부 수정 / 결론 수정 / 원문 보존율 조정 / 대중성 조정

수정 범위:
- 전체 재작성 / 일부 재작성 / 문장 다듬기 / 구성안 재조정 / 제목·썸네일만 수정 / 오프닝만 교체

수정 방향:
-

수정 후 확인할 점:
-
```

수정본 출력:

```
# 수정본

## 수정 요약

- 바뀐 점:
- 유지한 점:
- 의도:

---

## 수정된 대본

[수정본]
```

## 12. 제목·썸네일 후보 템플릿

```
# 제목 후보

1. [질문형 제목]
   - 방향:
   - 장점:
   - 주의:

2. [역설형 제목]
   - 방향:
   - 장점:
   - 주의:

3. [차분한 교양형 제목]
   - 방향:
   - 장점:
   - 주의:

# 썸네일 문구 후보

1.
2.
3.

# 추천 조합

- 제목:
- 썸네일:
- 이유:
```

제목과 썸네일은 본문에서 회수 가능한 질문이어야 한다.

## 13. 최종 검수 템플릿

항상 길게 출력하지 않는다.  
위험이 있을 때만 짧게 알린다.

```
# 최종 검수

- 원문의 핵심 메시지가 유지되었는가:
- 원문의 강점이 살아 있는가:
- 원문과 반대되는 의미가 되지 않았는가:
- 중심 질문이 영상 끝에서 회수되는가:
- 오프닝만 강하고 본문이 약하지 않은가:
- 말로 들었을 때 문장이 자연스러운가:
- 전문 용어가 필요한 만큼 설명되었는가:
- 감성이 논리를 대신하고 있지 않은가:
- 제목과 썸네일이 본문을 배반하지 않는가:
- 외부 조사가 필요한 내용인데 확인 없이 단정하지 않았는가:
- 요청에 비해 절차가 과하게 출력되지 않았는가:
```
